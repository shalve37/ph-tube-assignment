Answers of the blog questions

  • Discuss the scope of var, let, and const

    Answer:

    var: var is the oldest of three variable declaration methods. Those three are: var, let and const(constant). Before ES6 var was very popular. var is globally scoped or function scoped. Globally scoped means the var is declared outside of any function and this var can be accessed throughout the code. Function scoped means that the var is declared inside a function and can only be accessed inside the function.


    let: let is a block scoped variable declaration method. Block scoped means it works within a {}(curly braces). Whatever is inside the curly braces is called a block. Let can only be accessed inside this block. Let can be updated inside its block, but cannot be re-declared. You can declare the same let in another block.


    const: const(constant) is also a block scoped variable. It can only be accessed inside the block its declared. Const cannot re-assigned or re-declared within the block.

  • Tell us the use cases of null and undefined

    Answer:

    null: null is an assignment value. When you set a variable's value to 'null' it means that the variable has no value or the variable is empty. It is done intentionally.


    undefined: undefined simply means something that is not defined. It is something that javascript produces when running the code. When you declare a variable but don't assign any value to it. When you don't return a value in function it shows undefined when the function is called.

  • What do you mean by REST API?

    Answer:

    An API(Application Programming Interface) is a set of rules that tells how the devices or applications can connect and communicate with each other. It works between two software or devices. A REST(Representational State ) API is a set of rules for creating and interacting with websites and applications. It is an architectural style for designing networked applications.
